# Phase 4: Output & App Polish Implementation Log

## Overview
In Phase 4, we focused on finalizing the application loop by allowing users to export their segmented captions. We successfully implemented an export pipeline that converts our internal `CanonicalTimeline` into standardized caption formats (SRT, VTT) alongside raw JSON, preserving word modifications made directly in the UI.

## What We Did
1. **Export Engine (`src/pipeline/export/exporter.py`)**: 
   - Created the `TimelineExporter` handling `CanonicalTimeline` to SRT and WebVTT formatting.
   - Built custom formatters for `HH:MM:SS,mmm` (SRT) and `HH:MM:SS.mmm` (VTT) derived right out of the seconds-based float timing.
2. **UI Integration (`src/ui/windows/main_window.py`)**:
   - Transformed the placeholder "Export JSON" button into an active "Export Subtitles" workflow.
   - Connected `btn_export` directly to `TimelineExporter`.
   - Wired the UI to dynamically read user modifications from the `QTableWidget` to update the `CanonicalTimeline` object before writing, ensuring all manual fixes are preserved on export. 
   - Linked to `QFileDialog` natively dropping assets directly into the project's internal `exports/` folder default pathway.

## Difficulties & Hard Things
1. **Time Math Simplification:** Subtitle formats dictate very specific string layouts for their timecodes (e.g. SRT uses a comma for milliseconds, while VTT uses a period). Translating floating-point seconds into zero-padded Hours, Minutes, Seconds, and Milliseconds required careful `.zfill()` and modulo division logic to prevent synchronization drifts natively in Python.
2. **State Synchronization:** In PySide6, visually rendering text on a `QTableWidget` strips it away from our base Pydantic schema context. Re-coupling these involved iterating strictly over the visible table rows and injecting those user-amended strings back into our Pydantic `segments[row].text` field just prior to writing the file to disk.

## Assumptions Made
1. **Export Validations:** The implementation assumes that the "Export Subtitle" button is strictly deactivated until a full `CanonicalTimeline` completes transcription. It operates on the condition that `self.current_timeline_path` is completely generated and physically saved to disk first. 
2. **Edits are Segment-Level:** The app currently writes any user-made text modification into the broader `segment.text` field. It assumes that correcting typos does not functionally alter the individual precise word-level start/end timings generated by WhisperX, but perfectly preserves the caption-level presentation.

## What Went Wrong & How We Fixed It
1. **Nested Directory Collision:** While running terminal commands to create the `__init__.py` file for the new `export` module, a pathing error accidentally created a recursive `desktop-app/desktop-app/` nested folder tree.
    * *Fix*: Safely caught the folder structure duplication immediately, ran `Remove-Item -Recurse -Force desktop-app` to prune the accidental ghost folder, and regenerated the correct paths cleanly before touching the UI.
2. **Package Deprecation Warning:** During the final PySide6 syntax check, Python warned us that `google.generativeai` package support natively ended, favoring the new standard `google.genai`.
    * *Fix/Note for Future*: Though the app perfectly compiles for now, next iterations will gracefully upgrade that prompt wrapper to the latest supported Google library.

## Modifications to Initial Plan
*   **Upgrading Export Beyond JSON:** Originally, the plan focused largely on exporting just the `CanonicalTimeline` raw JSON dataset. We pivoted mid-flight to immediately wire the `SRT` and `VTT` processors to that UI button as well, recognizing that a user natively completing timeline-fixes needs a usable industry-standard end product without diving through an external converter script.

## Important Takeaways
The **CaptionLayer Desktop App MVP** is feature complete. The timeline pipeline flows smoothly through local instantiation -> WhisperX -> Rule Based chunking -> Gemini Casing/Punctuation -> Timeline Viewer Edits -> Multi-format subtitle outputs. All modules sit comfortably behind PySide6's background `QThread` ensuring visual stability.
